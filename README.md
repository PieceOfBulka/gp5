# Поставновка задачи
Каждый день на музыкальную стриминговую площадку Spotify загружается несколько десятков тысяч песен. Это вылевается в два челленджа для такой платформы:
- большие ручные ресурсы нужны, чтобы обрабатывать такой объем треков
- разметка новых песен является субъективной оценкой сотрдуника-разметчика

Музыкальный стриминг хотел бы, чтобы обе части его главных ца - слушатели и исполнители были довольны.
Для этого надо первым рекомендовать песни вторых, которые им понравятся и они будут их переслушивать, а следовательно все чаще заходить в сервис.

И музкальной площадке хотелось бы заранее знать, какие из добавляющихся песен станут популярными - то есть большая масса людей будет слушать ее на репите.
Для ручного анализа фактора популярности надо иметь в штате не просто сотрудников, а экспертов (которым надо платить немаленькую зп).
Хочется автоматизировать данный процесс - тем более большинство популярных песен имеет похожий паттерн, который как раз может распознать нейросеть и предсказать популярность.
Мы снизим нагрузку (только частично, а не полностью - важно проверять результаты программы) на экспертов, а также повысим шанс локальных/малозаметнх треков (но с большим потенциалом на успех) выбиться в топ и быть замеченными экспертами.

Более того, всегда большой проблемой была разметкой жанров - одного из главных факторов, который влияет на желание человек слушать этот трек.
Жанр часто проставлен на уровне исполнителя (в котором он чаще всего поет или сам выбрал) - особенно у малопопулярных исполнителей.
Это ухудшает работу рекомендаций и поиска. Ручная разметка ресурснозатратна (дорогая и долгая) и может быть субъективной, неточной.

Поэтому данный проект поделен на две части (две задачи):
1) Предсказание популярности трека (оценка от 0 до 100) по табличным данным (полносвязная сеть)
2) Предсказание жанров по текстам песен (многоклассовая классификация)

# Данные
Оба датасета были  взяты с платформы Kaggle и обработаны для обучения моделей DL

Задача 1: https://www.kaggle.com/datasets/saichaitanyareddyai/spotify-tracks-dataset-audio-features?resource=download

Задача 2: https://www.kaggle.com/datasets/evabot/spotify-lyrics-dataset

Также для расширения списка песен в 2 задаче (после чистки их осталось практически в 2 раза меньше, чем было изначально) команда использовала официальные API Spotify и Genius для получения дополнительной информации по добавленным песням (текст песни + жанр)

# Модели предсказания

1) Задача предсказания популярности треков
- FCNN (ветка task_1) с проведенным заранее EDA (анализ датасета + очистка и нормализация)

2) Задача классификации жанров по тексту песни
- baseline: TF-IDF и многоклассовая классификация oneVSall
- 3 архитектуры DL:
  - BiLSTM
  - CNN text
  - fine-tuning BERT
